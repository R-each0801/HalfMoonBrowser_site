# 13. 広告ブロックの詳細設定

## 広告ブロックの仕組み

HalfMoon の広告ブロックは以下の **3 層構造** です:

| 層 | 技術 | 説明 |
|---|---|---|
| 1. URL フィルタ | WKContentRuleList | 217 のルールで広告配信 URL をブロック |
| 2. CSS 非表示 | JavaScript 注入 | 広告ネットワークの id/class を CSS で消す |
| 3. 強力ブロック | JavaScript 注入 | アンチ広告ブロック検知バイパス、無限リロード対策など |

## 2 つのトグル

### 「広告ブロック」

ON で **層 1 + 層 2** が有効になります。通常はこれだけで十分。
OFF にすると一切のブロックを行いません。

### 「強力ブロック」

ON で **層 3** も追加で有効になります。「広告ブロックを無効にしないと見られません」
系のサイトに効きます。**ただし一部サイトでレイアウト崩れや動画再生不可などの
副作用が出ることがあります。**

## サイト別 ON/OFF

表示が崩れるサイトだけ広告ブロックを OFF にしたいときは、ダイヤルに
**「サイト別広告ブロック」** を追加しておくと、そのサイトでワンタップで切替え
できます。

管理は:

- 設定 → 広告ブロック → **「サイト別の広告ブロック」**

OFF にしたサイトの一覧が出て、そこから復元できます。

## カスタムフィルタ

自分で URL パターンを追加してブロックしたい広告配信ドメインがあれば:

- 設定 → 広告ブロック → **「カスタムフィルタ」** → ＋ ボタンで追加

例: `example-ads.com` → この文字列を含むリクエストはすべてブロック。

## 消した広告の管理

[07. 広告を消す](07-practice-adblock-select.md) で **自分で非表示にした要素** の
一覧・復元はここから:

- 設定 → 広告ブロック → **「消した広告の管理」**

サイトごとにセクション分けされ、各セレクタを 1 つずつ削除 / サイト全体で戻す
ことができます。

## 特殊なサイトの扱い

いくつかのサイトは検知が特に厳しく、普通に広告ブロックすると
**白紙ページになる** ことがあります。HalfMoon はこれらを内部的に
以下の扱いに分類しています:

- **Exempt (完全除外)**: 広告ブロック OFF と同じ扱い（例: x.com の UI が壊れる問題）
- **CSS-only**: URL ブロックは行わず、CSS での非表示のみ（例: oricon.co.jp）
- **anti-adblock**: 強力ブロックのみで検知回避スクリプトを適用（例: 一部の livedoor 系まとめ）

これらはアプリ内で自動判別されるので、ユーザー操作は不要です。もし
「このサイトが開けない」ものを発見したら、[18. FAQ](18-faq.md) を参照してください。
